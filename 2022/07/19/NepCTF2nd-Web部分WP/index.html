获取 admin 的密码

  • 这题肯定是要用 admin 登录的，所以方式不外乎几点，通过提权修改 admin 的密码；或者通过注入爆出 admin 的密码。

根据源码在 login.php 的 28-30 行，用户登录后 session 存储了 username 和 studentid 的源数据

之后在 index.php 的 183-184 行可以看到在存储用户答案的时候，没有对传入的参数($_SESSION['username'])进行过滤，当时没想到居然是这个地方的注入，氦，一直在登录和注册的地方试，太蠢了。

$query = "insert into score values (NULL, '".addslashes_deep($_SESSION['username'])."', $getscroe);
insert into userAnswer values (NULL, '".$_SESSION['username']."', '".$_POST['q1']."', '".$_POST['q2']."', '".$_POST['q3']."', '".$_POST['q4']."', '".$_POST['q5']."')";

这里由于一点点小问题导致没有对 q1-5 进行合法性验证，所以导致在这里即可直接使用堆叠注入进行下一步操作，这里的考点是二次注入，利用 username 进行堆叠注入，修改 admin 的 studentid，从而登录获取 admin。现题目已经修改完毕，增加了对 q1-5 的合法性验证，附件也将会同步更新。

因为通篇来看，只有这处未对 username 转义，而且此处进行了 multi_query，所以可以如果此处存在注入点的话就可以进行堆叠注入。

首先尝试闭合，可以看到如果我的 username 为

a','1','2','3','4','1');update users set studentid='{passwd}' where username='admin';

那么执行的 sql 语句就变成了这样

...;insert into userAnswer values (NULL, 'a','1','2','3','4','1');update users set studentid='admin_passwd' where username='admin';#', '1', '1', '1', '1', '1')";

执行后就更改了 admin 的 studentid 为 admin_passwd，即可用 admin 登录。

类加载与反序列化

结合附件中的源码文件，正好想到了 Assembly 远程类加载。关键就在于如何触发到此类的加载。由于文件中说了不涉及到业务，因此需要特殊的办法来加载。将请求打乱传入 $type 得到的报错中能得知题目使用了 Json.net

尝试反序列化加载类，会发现使用原本的 Payload 会报错，就是并没有所需要的类。配合任意文件写入能想到需要写入一个类，然后和附件中给出的类连接起来一起用。而 Assembly 只需要类名，加载后就会 CreateInstance，因此将恶意代码构造在类的构造函数中。

namespace rceTest3{
